# Subfield / tag vocabulary

Use these labels when naming clusters so scope lines up with the pipeline's paper
tags (SPEC §3b). Not exhaustive — coin a precise new label when none fits, but
prefer an existing one to keep clusters and tags aligned.

## Application (biological problem) — good cluster names

- Variant effect prediction
- Perturbation prediction
- Spatial domains / spatial transcriptomics
- Gene regulatory network (GRN) inference
- Multiomics integration
- Drug response
- Cell annotation
- Genome annotation
- Clinical genetics
- CRISPR screens
- Protein structure / function
- Single-cell foundation models

## Method (computational class) — good keyword seeds

- Optimal transport
- DNA / RNA language model
- Foundation model
- Graph neural network
- CNN + Transformer
- Benchmark / metric
- Dataset
- Software toolkit / pipeline
- Generative / diffusion / ODE
- Gradient-boosted trees
- Saturation genome editing (MAVE)
- Prime editing
- U-Net, RetNet, state-space models

## Naming guidance

- Cluster **name** = an *Application* (what biological question), when possible.
- Cluster **keywords** = a mix of *Method* terms + specific assays/data types the
  subfield uses. This is what the per-cluster research agent searches on.
